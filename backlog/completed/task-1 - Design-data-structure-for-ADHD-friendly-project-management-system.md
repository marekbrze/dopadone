## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Design hierarchical data model for ADHD-friendly project management system using Domain-Driven Design principles.

## Domain Model

### Bounded Context: Project Management

**Hierarchy:**
- AREAS: Top-level organization (Work, Home, Personal Projects)
- SUBAREAS: Second-level, children of Areas (Home → Car, Work → Marketing)
- PROJECTS: Goal-oriented task groups within Subareas, recursively nestable

**Example:**
Area: Home → Subarea: Travel → Project: Trip to Japan → Sub-projects: Hotel booking, Flight tickets, Itinerary planning

### Entities

**Area** (Aggregate Root)
- id: UUID PRIMARY KEY
- name: TEXT NOT NULL
- color: TEXT (hex code, nullable)
- created_at: TIMESTAMP NOT NULL DEFAULT NOW
- updated_at: TIMESTAMP NOT NULL DEFAULT NOW
- deleted_at: TIMESTAMP NULL (soft delete)

**Subarea** (Aggregate Root)
- id: UUID PRIMARY KEY
- name: TEXT NOT NULL
- area_id: UUID NOT NULL → areas.id
- color: TEXT (hex code, nullable, inherits from parent Area if null)
- created_at: TIMESTAMP NOT NULL DEFAULT NOW
- updated_at: TIMESTAMP NOT NULL DEFAULT NOW
- deleted_at: TIMESTAMP NULL (soft delete)

**Project** (Aggregate Root, recursively nestable)
- id: UUID PRIMARY KEY
- name: TEXT NOT NULL
- description: TEXT (optional, markdown support)
- goal: TEXT (optional - "what I want to achieve")
- status: ENUM('planned', 'in_progress', 'completed', 'archived', 'paused', 'abandoned') NOT NULL DEFAULT 'planned'
- priority: ENUM('high', 'medium', 'low') NOT NULL DEFAULT 'medium'
- progress: INTEGER CHECK (progress >= 0 AND progress <= 100) DEFAULT 0
- start_date: DATE (optional)
- deadline: DATE (optional)
- color: TEXT (hex code, nullable)
- parent_id: UUID NULL → projects.id (self-reference for nesting)
- subarea_id: UUID NULL → subareas.id (required for root projects, NULL for nested)
- position: INTEGER DEFAULT 0 (user-controlled ordering)
- created_at: TIMESTAMP NOT NULL DEFAULT NOW
- updated_at: TIMESTAMP NOT NULL DEFAULT NOW
- completed_at: TIMESTAMP NULL
- deleted_at: TIMESTAMP NULL (soft delete)

**Constraints:**
- CHECK: (parent_id IS NOT NULL) OR (subarea_id IS NOT NULL) -- project must have either parent or subarea
- No UNIQUE constraint on name (names can repeat)

### Value Objects (in Go code)

- Color: validated hex string
- ProjectStatus: enum type
- Priority: enum type
- DateRange: start_date + deadline with validation (start < deadline)

### Indexes

- projects: idx_projects_deadline (deadline) -- date-based queries
- projects: idx_projects_status_priority (status, priority) -- filtering
- projects: idx_projects_parent_id (parent_id) -- nested project lookup
- projects: idx_projects_subarea_id (subarea_id) -- root project lookup
- subareas: idx_subareas_area_id (area_id)
- All: idx_*_deleted_at (deleted_at) -- exclude soft-deleted

### Tech Stack

- **Database**: SQLite (local file, zero configuration)
- **Migrations**: Goose (https://github.com/pressly/goose)
- **Query Generation**: sqlc (https://github.com/kyleconroy/sqlc)
- **Timezone**: Local system time (no timezone conversion)

### File Structure

```
migrations/
  20240301000000_initial_schema.up.sql
  20240301000000_initial_schema.down.sql
queries/
  areas.sql
  subareas.sql
  projects.sql
internal/
  db/
    models.go      (generated by sqlc)
    queries.sql.go (generated by sqlc)
  domain/
    area.go
    subarea.go
    project.go
    value_objects.go
```

### Notes

- Soft delete pattern: deleted_at IS NULL means active record
- Recursive project nesting via parent_id self-reference
- User can choose sort mode: manual (position) or auto (deadline/priority)
- Completed_at set when status changes to 'completed'
- Future: Tasks entity (separate task, not in scope)
