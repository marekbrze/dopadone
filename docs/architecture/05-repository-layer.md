# Repository Layer

## Overview

The Repository Layer provides data access abstraction using the **Repository Pattern**. It uses **sqlc** for type-safe SQL query generation and implements the `db.Querier` interface for dependency injection.

**Key Characteristics**:
- **sqlc-generated**: Type-safe queries from SQL files
- **Interface-based**: `db.Querier` interface for testability
- **Transaction support**: TransactionManager for multi-entity operations
- **Soft delete pattern**: Entities marked as deleted, not removed

---

## db.Querier Interface

### Interface Definition

The Querier interface defines all database operations:

```go
// internal/db/querier.go (generated by sqlc)
type Querier interface {
    // Area operations
    CreateArea(ctx context.Context, arg CreateAreaParams) (CreateAreaRow, error)
    GetAreaByID(ctx context.Context, id string) (GetAreaByIDRow, error)
    ListAreas(ctx context.Context) ([]ListAreasRow, error)
    UpdateArea(ctx context.Context, arg UpdateAreaParams) (Area, error)
    HardDeleteArea(ctx context.Context, id string) error
    
    // Project operations
    CreateProject(ctx context.Context, arg CreateProjectParams) (Project, error)
    GetProjectByID(ctx context.Context, id string) (Project, error)
    ListProjectsBySubarea(ctx context.Context, subareaID sql.NullString) ([]Project, error)
    ListProjectsByParent(ctx context.Context, parentID sql.NullString) ([]Project, error)
    UpdateProject(ctx context.Context, arg UpdateProjectParams) (Project, error)
    HardDeleteProject(ctx context.Context, id string) error
    
    // Task operations
    CreateTask(ctx context.Context, arg CreateTaskParams) (Task, error)
    GetTaskByID(ctx context.Context, id string) (Task, error)
    ListTasksByProject(ctx context.Context, projectID string) ([]Task, error)
    UpdateTask(ctx context.Context, arg UpdateTaskParams) (Task, error)
    HardDeleteTask(ctx context.Context, id string) error
    
    // Count operations
    CountProjectsByArea(ctx context.Context, areaID string) (int64, error)
    CountTasksByProject(ctx context.Context, projectID string) (int64, error)
    
    // ... more methods
}
```

### Implementation

The `Queries` struct implements the interface:

```go
// internal/db/db.go
type Queries struct {
    db DBTX
}

func (q *Queries) CreateTask(ctx context.Context, arg CreateTaskParams) (Task, error) {
    row := q.db.QueryRowContext(ctx, createTaskQuery,
        arg.ID, arg.ProjectID, arg.Title, arg.Description, ...)
    // ... scan row into Task
}
```

```sql
-- internal/db/queries/tasks.sql

-- name: CreateTask :one
INSERT INTO tasks (
    id, project_id, title, description, start_date, deadline,
    priority, context, estimated_duration, status, is_next,
    created_at, updated_at, deleted_at
) VALUES (
    ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?
) RETURNING *;

-- name: GetTaskByID :one
SELECT * FROM tasks
WHERE id = ? AND deleted_at IS NULL;

-- name: ListTasksByProject :many
SELECT * FROM tasks
WHERE project_id = ? AND deleted_at IS NULL
ORDER BY created_at DESC;

-- name: UpdateTask :one
UPDATE tasks
SET 
    title = ?,
    description = ?,
    status = ?,
    priority = ?,
    updated_at = ?
WHERE id = ?
RETURNING *;

-- name: HardDeleteTask :exec
DELETE FROM tasks WHERE id = ?;
```

## Transaction Handling

### TransactionManager

```go
// internal/db/transaction.go
type TransactionManager struct {
    db *sql.DB
}

func NewTransactionManager(db *sql.DB) *TransactionManager {
    return &TransactionManager{db: db}
}

func (tm *TransactionManager) WithTransaction(ctx context.Context, fn func(Querier) error) error {
    tx, err := tm.db.BeginTx(ctx, &sql.TxOptions{
        Isolation: sql.LevelSerializable,
    })
    if err != nil {
        return fmt.Errorf("begin transaction: %w", err)
    }
    
    defer func() {
        if p := recover(); p != nil {
            tx.Rollback()
            panic(p)
        }
    }()
    
    txQuerier := &Queries{db: tx}
    if err := fn(txQuerier); err != nil {
        if rbErr := tx.Rollback(); rbErr != nil {
            return fmt.Errorf("rollback failed: %v, original error: %w", rbErr, err)
        }
        return err
    }
    
    if err := tx.Commit(); err != nil {
        return fmt.Errorf("commit transaction: %w", err)
    }
    
    return nil
}
```

### Usage in Service

```go
// internal/service/area_service.go
func (s *AreaService) HardDelete(ctx context.Context, id string) error {
    return s.txManager.WithTransaction(ctx, func(q db.Querier) error {
        // Delete children first (tasks → projects → subareas → area)
        if err := q.DeleteTasksBySubareaID(ctx, sql.NullString{String: subareaID, Valid: true}); err != nil {
            return fmt.Errorf("delete tasks: %w", err)
        }
        
        if err := q.DeleteProjectsBySubarea(ctx, id); err != nil {
            return fmt.Errorf("delete projects: %w", err)
        }
        
        if err := q.DeleteSubareasByArea(ctx, id); err != nil {
            return fmt.Errorf("delete subareas: %w", err)
        }
        
        if err := q.HardDeleteArea(ctx, id); err != nil {
            return fmt.Errorf("delete area: %w", err)
        }
        
        return nil
    })
}
```

**Benefits**:
- **Atomicity**: All operations succeed or all fail
- **Consistency**: Database remains consistent on error
- **Automatic rollback**: On error or panic

## Soft Delete Pattern

### Database Schema

```sql
CREATE TABLE tasks (
    id TEXT PRIMARY KEY,
    -- ... other fields
    deleted_at TIMESTAMP,
    CHECK (deleted_at IS NULL OR deleted_at <= CURRENT_TIMESTAMP)
);
```

### Query Pattern

All list/get queries filter out soft-deleted records:

```sql
-- name: GetTaskByID :one
SELECT * FROM tasks
WHERE id = ? AND deleted_at IS NULL;

-- name: ListTasksByProject :many
SELECT * FROM tasks
WHERE project_id = ? AND deleted_at IS NULL
ORDER BY created_at DESC;
```

### Soft Delete Implementation

```go
-- name: SoftDeleteTask :one
UPDATE tasks
SET deleted_at = CURRENT_TIMESTAMP, updated_at = CURRENT_TIMESTAMP
WHERE id = ?
RETURNING *;
```

## Common Query Patterns

### Get By ID

```go
func (s *TaskService) GetByID(ctx context.Context, id string) (*domain.Task, error) {
    res, err := s.repo.GetTaskByID(ctx, id)
    if err != nil {
        if errors.Is(err, sql.ErrNoRows) {
            return nil, ErrTaskNotFound
        }
        return nil, err
    }
    result := converter.DbTaskToDomain(res)
    return &result, nil

```

### List All

```go
func (s *TaskService) ListAll(ctx context.Context) ([]domain.Task, error) {
    dbTasks, err := s.repo.ListAllTasks(ctx)
    if err != nil {
        return nil, fmt.Errorf("list all tasks: %w", err)
    }
    
    tasks := make([]domain.Task, len(dbTasks))
    for i, dbTask := range dbTasks {
        tasks[i] = converter.DbTaskToDomain(dbTask)
    }
    
    return tasks, nil
}
```

## Testing Repository

Repository tests are **integration tests** using a real SQLite database:

```go
// internal/db/tasks_test.go
func TestCreateTask(t *testing.T) {
    // Setup test database
    db := setupTestDB(t)
    defer db.Close()
    
    querier := db.New(db)
    
    // Create test data
    area, _ := querier.CreateArea(ctx, db.CreateAreaParams{
        Name: "Test Area",
    })
    
    subarea, _ := querier.CreateSubarea(ctx, db.CreateSubareaParams{
        Name:   "Test Subarea",
        AreaID: area.ID,
    })
    
    project, _ := querier.CreateProject(ctx, db.CreateProjectParams{
        Name:      "Test Project",
        SubareaID: sql.NullString{String: subarea.ID, Valid: true},
    })
    
    // Test task creation
    task, err := querier.CreateTask(ctx, db.CreateTaskParams{
        ID:        uuid.New().String(),
        ProjectID: project.ID,
        Title:     "Test Task",
        Status:    "todo",
        Priority:  "medium",
    })
    
    if err != nil {
        t.Fatalf("failed to create task: %v", err)
    }
    
    if task.Title != "Test Task" {
        t.Errorf("expected title 'Test Task', got %s", task.Title)
    }
}
```

## Best Practices

### 1. Use sqlc, Not Raw SQL

❌ **Don't**:
```go
row := db.QueryRow("SELECT * FROM tasks WHERE id = ?", id)
```

✅ **Do**:
```go
task, err := querier.GetTaskByID(ctx, id)
```

### 2. Always Check for sql.ErrNoRows

```go
task, err := querier.GetTaskByID(ctx, id)
if err != nil {
    if errors.Is(err, sql.ErrNoRows) {
        return nil, ErrTaskNotFound
    }
    return nil, err
}
```

### 3. Use Transactions for Multi-Entity Operations

```go
// ✅ Good: Atomic cascade delete
txManager.WithTransaction(ctx, func(q db.Querier) error {
    q.DeleteTasksByProject(ctx, projectID)
    q.DeleteProject(ctx, projectID)
    return nil
})

// ❌ Bad: Non-atomic operations
querier.DeleteTasksByProject(ctx, projectID)
querier.DeleteProject(ctx, projectID) // Could fail, leaving orphaned tasks
```

### 4. Filter Soft-Deleted in Queries

Always include `AND deleted_at IS NULL` in SELECT queries:

```sql
-- ✅ Good
SELECT * FROM tasks WHERE id = ? AND deleted_at IS NULL;

-- ❌ Bad (returns soft-deleted tasks)
SELECT * FROM tasks WHERE id = ?;
```
